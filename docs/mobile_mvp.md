# 移动端 MVP 与关键问题澄清

本文档用于明确移动端 MVP 功能范围与关键设计决策，确保实现无二义性。

## 目标与定位
- 移动端作为“轻量复习入口”，专注于“获取后端数据 → 展示 → 复习 → 回传结果”。
- 不在移动端实现复杂创建/编辑/AI 生成流程。
- 支持离线可读 + 延迟同步，保证弱网/断网可使用。

---

## MVP 功能清单（细化版）

### 1. 设备级用户标识
- **目标**：建立移动端与 Web 端的同一用户身份。
- **输入**：首次启动无 `userId`。
- **输出**：生成并持久化 `userId`（UUID），所有 API 请求带此标识。
- **验收标准**：卸载前多次启动 `userId` 不变；后端返回的内容与同一用户一致。

### 2. 词表列表
- **目标**：展示用户已有词表。
- **输入**：`GET /lists`（携带 `userId` 头或参数）。
- **输出**：展示列表名、单词数、最近更新时间。
- **验收标准**：列表数量和名称与 Web 端一致。

### 3. 词表详情
- **目标**：查看词表中单词明细。
- **输入**：`GET /lists/:id` 或同等详情接口。
- **输出**：单词、基础释义、例句至少 1 条。
- **验收标准**：进入词表可见完整词项，点击词项可进入详情页。

### 4. 单词详情页
- **目标**：集中展示单词信息用于复习。
- **输入**：`GET /words/:wordId`。
- **输出**：单词、音标（如有）、释义、例句、相似词（如有）。
- **验收标准**：详情页信息来源于后端已有数据（非实时生成）。

### 5. 开始复习（练习流入口）
- **目标**：一键进入练习流程。
- **输入**：`POST /learn/start` 或 `POST /quiz/start`，包含 `listId` 与 `userId`。
- **输出**：题目列表（至少 5 题或后端默认）。
- **验收标准**：点击“开始复习”后立即进入第一题。

### 6. 做题流程（题目渲染与作答）
- **目标**：完成题目作答并获取反馈。
- **输入**：题目结构（题干、选项、正确答案或校验接口）。
- **输出**：对错反馈与正确答案（如后端提供）。
- **验收标准**：可完成题组并进入结果页或继续下一组题目。

### 7. 学习进度写回
- **目标**：复习结果同步后端，用于后续题目分配。
- **输入**：答题结果（例如 `learnedPoint`、正确率、词条完成情况）。
- **输出**：调用更新接口写回后端。
- **验收标准**：再次复习时题目权重或顺序体现学习进度变化。

### 8. 发音播放
- **目标**：单词/句子音频可播放并复用缓存。
- **输入**：`/audio` 相关接口。
- **输出**：音频播放与本地缓存。
- **验收标准**：点击播放 2 秒内可听到发音，后续播放不重复拉取。

### 9. 基础缓存（离线可读）
- **目标**：断网情况下仍可浏览已访问内容。
- **输入**：词表、词详情、题目数据。
- **输出**：本地缓存（SQLite/本地存储）。
- **验收标准**：断网时可打开最近访问的词表与单词详情。

## 已确认关键问题与方案

### 1）App 与 Web 用户如何关联
- **方案**：绑定码/二维码。
- **流程**：
  1. Web 端生成绑定码/二维码（含 `userId` 与服务器地址）。
  2. App 扫码后保存 `userId` 作为本地身份。
  3. 之后所有请求携带 `userId`。

### 2）App 端轻业务逻辑
- **原则**：移动端只做展示、播放、提交结果。
- **后端负责**：题目生成、难度控制、AI 生成、学习进度计算、语音生成。

### 3）离线可读 + 延迟同步
- **下行同步**：进入页面或手动刷新时从后端拉取最新内容。
- **上行同步**：答题结果先本地队列缓存，网络恢复后统一上报。
- **冲突策略**：后端为准，客户端只提交增量结果。

---

## App 如何发现 Web 端（非云端，IP 变化）

### 采用方案：二维码绑定 + 可手动修改服务器地址
- **二维码内容**：包含后端地址（如 `http://192.168.1.8:3000`）与绑定码/令牌。
- **App 扫码后**：保存 `baseURL` 与 `userId`。
- **IP 变化时**：重新扫码或手动修改 `baseURL`。

### 备用方案（可选增强）
- **固定内网 IP**：在路由器上设置 DHCP 保留，降低变更频率。
- **局域网自动发现**：mDNS/Bonjour 自动发现（实现复杂，非 MVP）。

- App 提供“服务器地址设置页”，支持手动输入与一键重连。
- 缓存数据定期清理与版本更新策略。
